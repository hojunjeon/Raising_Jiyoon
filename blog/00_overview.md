# "김지윤 키우기" 개발기 — Claude Code 에이전트 팀으로 LLM 게임 만들기

이 시리즈는 LLM 기반 인터랙티브 게임 "김지윤 키우기"를 Claude Code 에이전트 팀으로 개발하는 과정을 기록한다.

---

## 이 게임은 무엇인가

**"김지윤 키우기"**는 전직 초등학교 국어 선생님 김지윤(29세)이 SSAFY 15기 서울 6반에서 SW 개발자로 성장하는 이야기다. 플레이어가 김지윤의 대사를 직접 입력하면 NPC들이 반응하고, 선택에 따라 언변/개발/기획 능력치가 달라진다. 능력치에 따라 강사, 개발자, 기획자 등 다양한 엔딩에 도달한다.

```
[메인화면]  씬 목록 + 능력치 바
    ↓
[씬 선택]   입과식 / 알고리즘 수업 / 팀 프로젝트 ...
    ↓
[채팅창]    플레이어가 "김지윤의 대사"를 입력
    ↓
[NPC 반응]  AI가 NPC 캐릭터로 응답 + 능력치 변화 + 다음 선택지
```

---

## 왜 Claude Code 에이전트 팀인가

MVP 마감이 5일 남은 상황에서 시작했다. 프론트엔드(React), 백엔드(FastAPI), 캐릭터 콘텐츠, AI 연동을 혼자 순서대로 만들면 기간 안에 끝낼 수 없다.

Claude Code에는 **에이전트 팀**을 구성할 수 있는 기능이 있다. 역할을 나눈 에이전트들이 독립적인 작업을 동시에 실행하는 구조다. 이걸 처음부터 제대로 세팅하고 시작하면 5일도 가능하다고 판단했다.

---

## 에이전트 팀 구조

```
[게임 오케스트레이터]  — 전체 Phase 조율, 작업 할당
  ├── [콘텐츠 작가]      — 캐릭터 파일, 시나리오 작성
  ├── [프론트엔드 개발자] — KakaoTalk 스타일 채팅 UI
  ├── [백엔드 개발자]    — FastAPI API, 게임 상태 관리
  └── [AI 통합 전문가]   — HuggingFace 모델 연동
```

오케스트레이터가 전체 Phase를 관리하고, 서로 의존성이 없는 작업(FE/BE/콘텐츠)은 팬아웃으로 병렬 실행한다. Phase 간 의존성(백엔드 완료 전 AI 연동 불가)만 순서를 지킨다.

에이전트 간 데이터 전달은 `_workspace/` 폴더 파일로 처리한다. 백엔드 개발자가 `_workspace/api-spec.md`를 쓰면 프론트엔드 개발자가 읽어서 연동하는 방식이다.

---

## 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 프론트엔드 | React + Vite | 빠른 개발 환경 |
| 백엔드 | FastAPI | Python 기반, HuggingFace 연동 용이 |
| LLM | HuggingFace 한국어 모델 | 캐릭터 말투 구현 |
| DB | SQLite | MVP 단계 단순화 |

LLM은 MVP 단계에서는 Mock으로 시작해 API 구조를 먼저 완성하고, 이후 실제 모델로 교체하는 전략을 썼다. `LLM_BACKEND` 환경변수 하나로 mock/claude/huggingface를 전환할 수 있게 설계했다.
