## 프로젝트와 마감의 현실

게임을 만들기로 결정했을 때, 시간이 정말 없었다. **코드가 전혀 없는 상태에서 MVP 마감이 5일 후였다.**

일반적인 순차 개발 단계는 이렇다:
1. 백엔드 API 설계 및 구현 (1~2일)
2. 프론트엔드 채팅 UI 구축 (1~2일)
3. 캐릭터 파일, 시나리오 작성 (1~2일)
4. LLM 모델 연동, 프롬프트 튜닝 (2~3일)

이걸 순서대로 하면 최소 6일, 실제로는 버그 수정과 통합 비용을 고려하면 2주 이상 걸린다.

그래서 처음부터 다르게 생각하기로 했다. **여러 에이전트(Claude Code의 팀 기능)를 동시에 가동하면 어떨까?**

---

## 무엇을 만들었나: 에이전트 팀 + 스킬 시스템

Claude Code `/harness` 메타스킬 한 번으로 다음을 자동으로 구성했다.

### 에이전트 5인팀

| 역할 | 담당 | 파일 |
|------|------|------|
| 오케스트레이터 | 전체 Phase 조율 및 팀 할당 | `orchestrator.md` |
| 콘텐츠 작가 | 캐릭터 파일, 시나리오 작성 | `content-writer.md` |
| 프론트엔드 개발자 | KakaoTalk 스타일 채팅 UI | `frontend-dev.md` |
| 백엔드 개발자 | FastAPI API, 게임 상태 관리 | `backend-dev.md` |
| AI 통합 전문가 | HuggingFace 모델 연동 | `ai-integrator.md` |

### 각 에이전트가 사용할 스킬 6개

- `character-writer` — 캐릭터 파일 자동 생성
- `scenario-writer` — 시나리오 씬 작성
- `chat-ui-builder` — React 채팅 UI 구축
- `game-api-builder` — FastAPI 게임 서버 구축
- `llm-integrator` — HuggingFace 통합
- `game-orchestrator` — 전체 개발 워크플로우 정의

각 에이전트는 자신의 영역에서 "캐릭터 만들어줘", "API 구현해줘" 같은 명령을 받으면 자동으로 해당 스킬을 호출하도록 설정되어 있다.

## 핵심 설계 결정: 감독자 + 팬아웃 구조

처음에는 에이전트들이 서로 통신하는 구조를 생각했다. 하지만 메시지 오버헤드, 의존성 추적, 지연 누적 때문에 복잡해진다.

대신 **감독자(Supervisor) + 팬아웃/팬인 혼합** 패턴을 택했다.

```
[오케스트레이터 - 감독자]
  ├─ Phase 1 (병렬 실행)
  │  ├── 콘텐츠_작가 (캐릭터 + 시나리오)
  │  ├── 프론트엔드_개발자 (채팅 UI)
  │  └── 백엔드_개발자 (API + 게임 상태)
  │
  └─ Phase 2 (백엔드 완료 후)
     └── AI_통합_전문가 (HuggingFace 연동)
```

**Phase 1에서는 3개 팀이 완전히 독립적으로 움직인다.** 캐릭터 파일, 프론트엔드 UI, 백엔드 API가 동시에 진행되고, 각자가 할 일만 한다.

**Phase 2는 백엔드가 완료되어야 시작할 수 있다.** LLM 통합은 API 엔드포인트가 정의되어야 구현할 수 있기 때문이다.

이렇게 하면 병렬성을 최대화하면서도 의존성은 최소화할 수 있다.

---

## 데이터 전달: 파일 기반 인터페이스

에이전트들이 직접 메시지를 주고받는 대신, **`_workspace/` 폴더에 파일을 남기는 방식**을 택했다.

예시:
- 백엔드 개발자가 `_workspace/api-spec.md` 작성
- 프론트엔드 개발자가 이 파일을 읽어서 API 엔드포인트 연동
- 콘텐츠 작가가 `_workspace/character-list.md` 읽어서 캐릭터 순서 파악

이 방식의 장점:
- **느슨한 결합**: 에이전트가 다른 에이전트의 상태를 직접 모를 필요 없음
- **비동기 진행**: 한 팀의 완료를 기다릴 필요 없고, 파일만 읽으면 됨
- **추적 용이**: 산출물이 파일로 남아서 히스토리 관리 쉬움

---

## 왜 FastAPI를 선택했나

백엔드를 FastAPI로 구축하기로 한 이유는 단순하다. **Python 생태계 통일**이다.

HuggingFace의 한국어 LLM은 Python 라이브러리다. 백엔드도 Python(FastAPI)이면:
- 모델 로딩, 프롬프트 생성, 응답 처리가 같은 환경에서 이루어짐
- 의존성 충돌 최소화
- 에이전트가 작성한 코드 간 통합이 자연스러움

Node.js + Python 조합도 가능했지만, 중간에 API 호출 레이어가 필요해지고 복잡도가 올라간다. 마감이 5일인 상황에서 불필요한 복잡성은 제거했다.

## Claude Code의 `/harness` 메타스킬이 마법을 한다

가장 놀라운 부분은 **에이전트 팀 자체를 자동으로 구성했다는 것**이다.

내가 한 일은:
- "이런 팀이 필요하다"는 요구사항을 설명
- 각 역할의 책임 범위 정의

그러면 `/harness`가:
- 각 에이전트 `.md` 파일 생성
- 스킬 frontmatter 작성
- `settings.json` 업데이트
- 트리거 조건 설정

을 자동으로 해준다.

가장 중요한 부분은 **트리거 조건**이다. 각 에이전트의 `description`에 "캐릭터 만들어줘", "API 구현해줘" 같은 문구를 명시해두면, 내가 그 말을 던질 때 자동으로 해당 에이전트와 스킬이 호출된다.
