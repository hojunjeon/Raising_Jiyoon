# 시나리오 작성 가이드

## 파일 위치

모든 시나리오 파일은 `Chat_Game/scenario/scene_{N}.md`에 저장. N은 1부터 순서대로.

## 필수 섹션 구조

```markdown
# Scene {N}: {씬 제목}

## 메타데이터
- **트리거 조건**: [이전 씬 완료 / 특정 능력치 도달 / 날짜 이벤트]
- **등장 캐릭터**: {캐릭터명 목록}
- **배경**: {장소 설명}
- **예상 플레이 시간**: {분}

## 시나리오 요약
[이 씬에서 일어나는 일 2~3줄 요약]

## 씬 진행

### 도입부
[상황 설명 / 나레이션]

**{캐릭터명}**: "{대사}"

[플레이어 자유 입력 or 선택지]

### 선택지 (해당 시)
> **A. {선택지 텍스트}**
> → 능력치 변화: {언변/개발/기획} +{N}
> → 다음: [다음 대화 흐름 or scene_{N+1}]

> **B. {선택지 텍스트}**
> → 능력치 변화: {언변/개발/기획} +{N}
> → 다음: [다음 대화 흐름 or scene_{N+1}]

### 전개 (선택지 후 분기)

#### A 선택 시
[대화 흐름]

#### B 선택 시
[대화 흐름]

### 결말
[씬 마무리 상황]

## 씬 결과
- **다음 씬**: scene_{N+1} [트리거 조건]
- **능력치 변화 요약**:
  - 최대 획득 가능: 언변 +{}, 개발 +{}, 기획 +{}
  - 최소 획득 가능: 언변 +{}, 개발 +{}, 기획 +{}
- **특수 이벤트 플래그**: {해당 시}

## 이벤트 기록 (해당 시)
- [ ] 과목 평가: pass/fail 기록
- [ ] 월말 평가: pass/fail 기록
- [ ] 결석/지각 처리
```

## 이벤트 트리거 규칙 (CLAUDE.md 기반)

### 필수 이벤트 씬
| 이벤트 | 빈도 | 조건 |
|--------|------|------|
| 과목 평가 | 총 10회 | 커리큘럼 내 배분 |
| 월말 평가 | 총 6회 | 매월 말 |
| SW 역량 테스트 | 1회 | 중반부 |
| 결석/지각 이벤트 | 수시 | 선택지 결과 |

### 엔딩 분기 힌트 배치
- 언변이 높아질 선택지: 발표 자원, 질문 많이 하기, 팀 조율
- 개발이 높아질 선택지: 밤새 코딩, 추가 학습, 디버깅 도전
- 기획이 높아질 선택지: 아이디어 제안, 기획서 작성, 비즈니스 관점

## 씬 구성 가이드라인

- **scene_1**: 입과 첫날 (오리엔테이션, 동기 소개, 첫 인상 형성)
- **scene_2~5**: 첫 번째 과목 (알고리즘/파이썬 기초, 첫 과목 평가)
- **scene_6~10**: 두 번째 과목 + 첫 월말 평가
- **scene_11+**: 팀 프로젝트, 취업 준비, 최종 엔딩

## 작성 원칙

- **선택지는 최소 2개, 최대 4개**: 너무 많으면 피로감
- **자유 입력 씬도 포함**: LLM이 처리하는 자유 대화 구간
- **이분법 피하기**: "좋은 선택 vs 나쁜 선택"이 아니라 능력치 방향이 다른 선택
- **SSAFY 감성**: 실제 SSAFY 수강생이 공감할 상황 (아침 지각, 코드 안 돌아감, 팀 갈등 등)
